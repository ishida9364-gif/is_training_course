#総合解析トレーニングコース第一回課題

##課題1

'echo "this is a very very long command for practice"'

においては全て手動で入力した。

'echo "this is yet another very very long command for practice"'

においてはCtrl+Rで先ほどのコマンドの履歴を呼び出し、Meta+Bでカーソルを移動させた後に追加の単語を手動で入力した。

'echo "this is a command for practice"

においても先ほどと同様にCtrl+Rでコマンド履歴を呼び出し、Meta+Bでカーソルを移動させた後にCtrl+Wで不要な単語を削除した。


##課題2

bashと指定したときに呼び出される実行ファイルを表示するコマンド、

'which bash'

により

/usr/bin/bash

と返ってきた。確かにbashというファイルが存在することが分かる。さらに、

'which -a bash'

と入力すると

/usr/bin/bash
/bin/bash

と返ってきた。これはPATHに含まれるすべての実行ファイルを表示しており、確かに、はじめの場所がPATHに含まれていると分かる。


##課題3

コマンド'ls'はディレクトリ内のファイル名を列挙するコマンドである。
マニュアルによると'-h'オプションで、ファイルサイズを人間に読める単位 (human readable) で表示できる。


##課題4

-'ls no_such_file.txt'

と入力したところエラーメッセージ

ls: cannot access 'no_such_file.txt': No such file or directory

と表示された。これはそのようなファイル、ディレクトリが存在しないことを意味する。

-'not_a_real_command'

と入力すると

not_a_real_command: command not found

というエラーメッセージが表示された。これは該当のコマンドが存在しないことを意味する。

-'rm'

と入力すると

rm: missing operand
Try 'rm --help' for more information.

というエラーメッセージが表示された。

これはコマンドの対象となるオペランドが存在しないので、ヘルプを参照するよう伝える旨のエラーメッセージである。

-'cd /no/such/dir'

と入力すると

bash: cd: /no/such/dir: No such file or directory

というエラーメッセージが表示された。

これは実際には存在しないディレクトリを指定したため、それを伝えるメッセージである。
